# SwingTrader2026

Single-page static web app for tracking planned trade setups.

## Architecture

- `data.js` — hand-edited `SETUPS` array (one entry per planned trade)
- `prices.js` — auto-generated by `fetch_setups.py`; daily OHLC bars per symbol
- `script.js` — derives trigger date, daily P&L, "if held to today" P&L from
  the two arrays above; renders the year heatmap, setup cards, history table,
  and detail modal
- `fetch_setups.py` — pulls daily OHLC from Yahoo Finance for each symbol in
  `data.js`, from `addedDate − 14 days` through today, and writes `prices.js`
- Service worker `sw.js` caches the static assets including `prices.js`
  (bump `CACHE_NAME` on releases)

## Setup data model

A setup row carries the plan (`symbol`, `direction`, `entryTrigger`, `stop`,
`target`, `addedDate`, `notes`, `tier`, `setupType`) plus the manual outcome
(`status`, `triggeredDate`, `closedDate`, `closePrice`, `closeReason`). All
P&L is computed at runtime from `prices.js`.

Trigger detection: walk bars from `addedDate` forward; first bar whose
`high ≥ entryTrigger` (Long) or `low ≤ entryTrigger` (Short) becomes the
trigger day. `triggeredDate` in `data.js` overrides this.

Closes are manual — set `status: 'closed'`, `closedDate`, optional
`closePrice`, and `closeReason` on the setup.

## fetch_setups.py — Setup & Usage

Pulls OHLC history needed by the app.

### First-time setup (Mac)

Requirements: Python 3.10+, installed via Homebrew (`brew install python`).

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Run

```bash
source .venv/bin/activate
python3 fetch_setups.py
```

This regenerates `prices.js`. Commit it alongside `data.js` so the deployed
site stays consistent.

### Each new terminal session

```bash
cd SwingTrader2026
source .venv/bin/activate
python3 fetch_setups.py
```

### Notes

- `fetch_setups.py` parses `data.js` with a regex (no JS engine needed) to
  enumerate symbols and `addedDate`s. Add a setup to `data.js` and the next
  run picks it up automatically — no script edits needed.
- yfinance fetches one symbol at a time; the batch API is unreliable.
